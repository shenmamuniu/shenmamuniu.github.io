Jenkins打包Springboot项目并部署至Docker中

Jenkins
发布日期:   2019-11-13
更新日期:   2020-04-30
文章字数:   772
阅读时长:   3 分

本文记录Jenkins根据每次git的合并自动打包springboot项目，并将项目自动部署到容器中。

运行环境

  1. Jenkins
  2. 源代码管理工具：gitblit

Jenkins的插件安装

Jenkins需要安装的插件：

  1. Git plugin
  2. Maven Integration plugin
  3. Publish Over SSH

Publish Over SSH 插件配置

在jenkins服务器上生成秘钥对,使用如下命令

ssh-keygen -t rsa

生成完毕之后会在~/.ssh目录下有私钥id_rsa和公钥id_rsa.pub两个文件
注意：一定要写passphrase，否则会连不上的

复制公钥内容

将公钥id_rsa.pub文件中的内容copy到应用服务器中的~/.ssh/authorized_keys文件中，如果此文件中已经有内容，换行追加即可，没有的话自行创建

Jenkins配置

在系统管理->系统设置->Publish Over SSH中配置参数

注意：其中Remote Directory是一个通过ssh工具传输文件时，应用服务器接收文件时的存储路径

相关参数说明：

  1. Source files: 需要上传到应用服务器的文件，一般为jar、war等编译后的文件
  2. Remove prefix：First part of the file path that should not be created on the remote server.

[附:]将本地的项目部署到docker中去的脚本

此脚本的主要执行过程：根据容器的镜像名称获取容器编号，然后停止并删除容器。接着根据镜像重启启动一个容器，并将应用的编译后的文件拷贝到容器的tomcat中，然后启动tomcat

# !/bin/bash
CONTAINER_ID=`docker ps| grep "neil/openjdk:v0.4"|awk '{print $1}'`
echo CONTAINER_ID
if [ -n "$CONTAINER_ID" ]; then
    docker stop $CONTAINER_ID
    docker rm $CONTAINER_ID
else # 如果容器启动时失败了，就需要docker ps -a才能找到那个容器
    CONTAINER_ID=`docker ps -a | grep "neil/openjdk:v0.4" | awk '{print $1}'`
    if [ -n "$CONTAINER_ID" ]; then  # 如果是第一次在这台机器上拉取运行容器，那么docker ps -a也是找不到这个容器的
        docker rm $CONTAINER_ID
    fi
fi
# 重新启动一个容器
docker run -d -ti -p 9000:8080 neil/openjdk:v0.4
NEW_CONTAINER_ID=`docker ps| grep "neil/openjdk:v0.4"|awk '{print $1}'`
# 向tomcat目录拷贝文件
docker cp  /opt/jeisp-admin.war $NEW_CONTAINER_ID:/opt/tomcat/apache-tomcat-8.5.30/webapps/jeisp-admin.war
# 启动tomcat
docker exec $NEW_CONTAINER_ID bash -c  "/opt/tomcat/apache-tomcat-8.5.30/bin/startup.sh"
echo complete
