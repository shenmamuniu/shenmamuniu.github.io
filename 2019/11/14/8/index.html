Mysql主从复制

Mysql 数据库集群
发布日期:   2019-11-14
更新日期:   2020-04-04
文章字数:   633
阅读时长:   2 分

Mysql主从复制又叫Mysql主从同步，是构建数据库高可用集群架构的基础，它通过将一台主机的数据复制到其他一台或多台主机上，并重新应用日志(Relay log)中的SQL语句来实现复制功能。

运行环境

Ubuntu 16.04(3个虚拟机，其中一主两从)

Mysql版本：5.7.23

Master配置

开启二进制日志

vi /etc/mysql/mysql.conf.d/mysqld.cnf

添加以下部分
[mysqld]
log_bin=mysql-bin   //[必须]启用二进制日志
server-id=140       //[必须]服务器唯一ID，默认是1，一般取IP最后一段
Mysql默认同步所有的数据库，也可以指定同步或不同步哪些数据库，添加配置：

# 不同步哪些数据库
binlog-ignore-db = mysql
binlog-ignore-db = test
binlog-ignore-db = information_schema

# 只同步哪些数据库，除此之外，其他不同步
binlog-do-db = game

注意：这里无论是binlog—do-db还是binlog_ingore_db，如果需要设置多个数据库的时候，不要用逗号隔开，另起一行重新写即可，参考上面的binlog-ignore-db的写0法。

重启Mysql数据库

创建用于同步的账户并授权

mysql -u root -p 

mysql>CREATE USER 'username@%' [IDENTIFIED BY 'PASSWORD'] \g

mysql> grant all privileges on *.* to yyzc@"%" identified by "." \g
Query OK, 0 rows affected (0.00 sec)

mysql> flush privileges; \g
Query OK, 0 rows affected (0.00 sec)

查看状态

获取File及Position的信息

mysql>show master status \g
+------------------+----------+--------------+------------------+
| File             | Position | Binlog_Do_DB | Binlog_Ignore_DB |
+------------------+----------+--------------+------------------+
| mysql-bin.000004 |      308 |              |                  |
+------------------+----------+--------------+------------------+

Slave配置

开启二进制日志并重启Mysql

配置从服务器Slave及启动复制

其中master_log_file及master_log_pos从Master中获取

mysql>change master to master_host='192.168.245.140',master_user='mysync',master_password='123456',master_log_file='mysql-bin.000004',master_log_pos=308;  
mysql>start slave;    //启动从服务器复制功能

查看Slave的状态

其中：Slave_IO及Slave_SQL进程必须正常运行，即YES状态，否则都是错误的状态。

mysql> show slave status \g

同步遇到的问题

  1. 因为是虚拟机复制的，所以Mysql的uuid是相同的导致复制失败

    修改:/var/lib/mysql中的auto.cnf中的server-uuid即可
