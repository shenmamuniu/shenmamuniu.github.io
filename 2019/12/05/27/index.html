Nginx在Windows和Ubuntu下的基本命令

Nginx
Linux
发布日期:   2019-12-05
更新日期:   2020-06-19
文章字数:   1.1k
阅读时长:   4 分

本文主要记录Nginx在日常使用时用到的基本命令

Windows中运行Nginx

  1. 启动
    start nginx
  2. 停止
    nginx.exe -s stop
    nginx.exe -s quit
    注：stop是快速停止nginx，可能并不保存相关信息；quit是完整有序的停止nginx，并保存相关信息。
  3. 重新载入
    nginx.exe -s reload
    当配置信息修改，需要重新载入这些配置时使用此命令。

Ubuntu 16.04中运行Nginx

启动及退出

  1. 安装
    sudo apt-get install nginx
  2. 启动
    nginx
  3. 重新加载配置
    nginx -s reload
  4. 退出
    nginx -s stop
  5. 检查nginx配置文件
    nginx -t

反向代理

Ubuntu中nginx的配置文件在/etc/nginx目录下
通过nginx服务器中的80端口访问其他服务器中的非80端口的服务

修改nginx配置文件：

server
{
    listen 80;
    server_name *****;
    location / {
   		proxy_redirect off;
 		proxy_intercept_errors on;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header REMOTE-HOST $remote_addr;
        proxy_set_header X-Forwarded-For "$http_x_forwarded,$remote_addr";
        proxy_pass http://ip:port; 
    }
}

启用gzip压缩

修改配置文件http部分的gzip部分为以下：

http{
	...
    gzip on;
    gzip_types text/plain text/css text/javascript application/javascript
    ...
}

其他配置：

  1. gzip_com_level 4；# 默认1，可取[1~9],值越大，压缩率越大，对服务器cpu的消耗就越高
  2. gzip_min_length 2k; # 默认值: 0 ，不管页面多大都压缩,建议设置成大于1k的字节数，小于1k可能会越压越大。
  3. gzip_buffers 4 16k;
  4. gzip_vary on # 加个vary头，给代理服务器用的，有的浏览器支持压缩，有的不支持，所以避免浪费不支持的也压缩，所以根据客户端的HTTP头来判断，是否需要压缩

Nginx动静分离

把静态资源如js、css、png等交由Nginx处理，不再向tomcat发送请求,Location可根据需要进行配置：

  1. 所有提交给Nginx的请求的静态文件均由Nginx自身处理
  2. 只有匹配到固定路径的请求，Nginx才会自行处理
# 默认当前server下的所有对js、css文件的请求都会匹配到
location ~ .*\.(js|css)?$
{
    root /usr/local/webapps;
    expires 1h;
}
# 匹配static下的credit和images下的js及css文件
location ~* \/static\/(credit|images).*\.(js|css|ttf|woff|woff2)?$ {
root   D:/nginx/reg/;
index  test.png;
}

Nginx 配置404或500页面

server{
    ...
    error_page  404 403 500 502 503 504  /404.html;
    location = /404.html {
        root   /usr/share/nginx/html; # 错误页面的存放路径
    }
    ...
    
}

注意，此时只有Nginx抛出的404才能显示这个页面，其反向代理的错误页面则不会显示这个，如果想要其反向代理的页面抛出404，还需要添加：

location / CMS{
           proxy_intercept_errors on;

           proxy_pass      http://www;

           proxy_set_header HOST   $host;

           proxy_set_header X-Real-IP      $remote_addr;

           proxy_set_header X-Forwarded-FOR $proxy_add_x_forwarded_for;
       }

Nginx配置别名(超好用的咧)

location ~^/nginx {
alias E:/DApps/;
}

alias指定的目录是准确的，即location匹配访问的path目录下的文件直接是在alias目录下查找的；
即：http://xxx/nginx/a.html ：访问的就是E:/DApps/a.html,至于location这里的结尾要不要加/,应该是没区别的，但alias结尾是一定要加的。

Nginx+Tomcat配置https

有两种方式：1.同时配置https,2.Nginx配https,Tomcat不配

Tomcat和Nginx同时配置https

修改nginx.conf文件

server {
    listen 443;
    server_name localhost;
    ssl on;
    root html;
    index index.html index.htm;
    ssl_certificate   cert/214293719880562.pem;
    ssl_certificate_key  cert/214293719880562.key;
    ssl_session_timeout 5m;
    ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:ECDHE:ECDH:AES:HIGH:!NULL:!aNULL:!MD5:!ADH:!RC4;
    ssl_protocols TLSv1 TLSv1.1 TLSv1.2;
    ssl_prefer_server_ciphers on;
    location / {
        root html;
        index index.html index.htm;
    }
}

Nginx跨域配置

add_header Access-Control-Allow-Origin *;
add_header Access-Control-Allow-Headers X-Requested-With;
add_header Access-Control-Allow-Methods GET,POST,OPTIONS;

放在server-》location里面即可，此时只作用于当前location，当然放在http里面也是可以的。
