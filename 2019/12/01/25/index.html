Linux中的硬链接和软链接

Linux
Linux
发布日期:   2019-12-01
更新日期:   2019-12-01
文章字数:   707
阅读时长:   2 分

本文主要介绍linux系统的软链接和硬链接的使用和差异。

linux文件的权限标识符

drwxr-xr-x

第一位是文件类型，-为一般文件，d为目录，l为链接文件
剩余9位则是代表【owner权限】【group权限】【others权限】
r:读，w:写，x：可执行

软链接和硬链接

为了解决文件的共享使用
ll、ls -l：罗列出当前文件或目录的详细信息，含有时间、读写权限、大小、时间等信息，如下：

drwxr-xr-x 2 root root 4096 11月  7  2018 conf.d
-rw-r--r-- 1 root root 1077 11月  7  2018 fastcgi.conf
-rw-r--r-- 1 root root 1007 11月  7  2018 fastcgi_params
-rw-r--r-- 1 root root 2837 11月  7  2018 koi-utf

软链接：同windows的快捷方式很像，删除不影响源文件，但若被指向的原文件被删除，则相关软连接就变成了死链接；

硬链接：允许一个文件有多个有效路径名，作用对象为文件，只能在同一文件系统中的文件之间进行链接，之所以文件建立了硬链接就会防止数据误删，是因为文件系统的原理是，只要文件的索引节点(inode)还有一个以上的链接（仅删除了该文件的指向），只删除其中一个链接并不影响索引节点本身和其他的链接（数据的实体并未删除），只有当最后一个链接被删除后，此时如果有新数据要存储到磁盘上，被删除的文件的数据块及目录的链接才会被释放，空间被新数据暂用覆盖。

软硬链接的相关命令

创建软链接

ln -s source target
如：
ln -s /xxx/lntest/soft/default.txt test2
在当前目录创建一个test2执行default.txt

然后可以查看test2的文件类型为l
注意：软链接支持目录及文件；

删除软链接

同删除文件是一样的，使用rm命令即可

修改软链接

ln -snf [source ] [target]

创建硬链接

ln source target

注意：
1、硬链接不需要使用绝对路径；
2、只能对文件创建硬链接；
3、同一个源文件的所有硬链接文件必须在同一个硬盘、同一个分区里面
4、使用场景很少，不建议使用；

软链接同硬链接的差异

软链接是快捷方式，可以用在硬盘合理分配上；
硬链接是文件名字，一个文件的许多名字互为硬链接；
