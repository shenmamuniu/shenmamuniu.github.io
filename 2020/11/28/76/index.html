Centos8的一些基本配置

centos
Linux
发布日期:   2020-11-28
更新日期:   2020-11-28
文章字数:   814
阅读时长:   3 分

最近在搞centos8，把其常用操作及配置做个简单的记录，包括静态ip配置、gui的安装、防火墙配置、vnc配置等。

环境

os: centos8.2

配置静态IP

/etc/sysconfig/network-scripts
vi ifcfg-ens160

这里en(ethernet)后面可能是其它值，具体如下：

o:主板板载网卡，集成的是设备索引号

p:独立网卡，PCI网卡

s:热插拔网卡，usb之类的扩展槽索引号

后面的数字：MAC地址+主板信息计算得出的唯一序号

修改以下内容：

# 设置静态IP
BOOTPROTO="static"
# 开机启动
ONBOOT="yes"
# 设置IP地址
PREFIX=24
IPADDR=192.168.1.80
GATEWAY=192.168.1.1
NETMASK=255.255.255.0
DNS1=192.168.1.1

重启网卡

nmcli c reload ens160

GUI界面安装

这里如果使用的centos8.2 dvd版的安装镜像安装时，可以直接选择安装图形界面的，如果没有安装可以执行以下命令配置

yum groupinstall "Server with GUI" -y

systemctl get-default

systemctl set-default graphical.target

reboot

VNC Server安装及配置

执行以下命令安装

yum groupinstall "X Window System"
yum install gnome-classic-session gnome-terminal nautilus-open-terminal control-center liberation-mono-fonts
unlink /etc/systemd/system/default.target
ln -sf /lib/systemd/system/graphical.target /etc/systemd/system/default.target

reboot # 重启机器

yum install tigervnc-server -y

cp /lib/systemd/system/vncserver@.service /etc/systemd/system/vncserver@:1.service

修改配置文件,这里以test用户为例：

vim /etc/systemd/system/vncserver@:1.service
添加User=test
并将里面的<User>替换为test

修改完成之后

systemctl daemon-reload
# 设置密码
vncpasswd

启动及重启命令

systemctl enable vncserver@:1.service # 设置开机启动
systemctl start vncserver@:1.service # 启动vnc会话服务
systemctl status vncserver@:1.service # 查看nvc会话服务状态
systemctl stop vncserver@:1.service # 关闭nvc会话服务

接下来就可以通过vnc viewer远程连接centos8的图形界面了。

防火墙配置

centos8安装完成之后，防火墙默认是启动的，可以通过以下命令对防火墙进行控制

# 查看防火墙状态
systemctl status firewalld.service
# 停止
systemctl stop firewalld.service
firewall-cmd --zone=public --add-port=80/tcp --permanent   # 开放80端口
firewall-cmd --zone=public --remove-port=80/tcp --permanent # 删除开放的80端口
firewall-cmd --reload   # 配置立即生效
# 查看已经开放的端口
firewall-cmd --zone=public --list-ports

硬盘扩容

首先给硬盘分配空间，然后使用gparted对分区扩容

安装gparted

可以使用gparted.iso，也可以在系统内安装gparted应用，系统内安装gparted应用可通过以下命令：

sudo yum install epel-release
sudo yum install gparted

安装完成之后通过gparted修改分区大小，然后扩容逻辑分区大小：

# 查看逻辑分区名：
df -h
# 扩容，这里+2G指添加的容量
lvextend -L +2G /dev/mapper/cl-root
xfs_growfs /dev/mapper/cl-root
# 查看各个分区容量
lsblk
